# Como contribuir

## Fluxo de trabalho

1. Abra uma issue descrevendo a tarefa ou problema.
2. Crie uma branch a partir de `main`.
3. Faça alterações pequenas e focadas.
4. Teste as regras afetadas.
5. Abra um pull request explicando o que mudou.

## Nomes de branches

- `feature/nome-da-funcionalidade`
- `fix/nome-do-problema`
- `docs/nome-da-documentacao`
- `art/nome-do-asset`

## Commits

Preferimos mensagens no formato:

- `feat: add turn system`
- `fix: prevent invalid movement`
- `docs: update game rules`
- `art: add Kin prototype model`
- `test: cover portal movement`

## Regras para assets

- Não enviar builds e pastas geradas pela Unity.
- Usar Git LFS para arquivos binários grandes.
- Manter arquivos `.meta` junto dos respectivos assets.
- Registrar a autoria e a origem de assets externos.
- Não versionar conteúdo sem direito de uso.

## Segredos

Nunca publicar tokens, chaves de API, senhas, certificados ou credenciais de serviços multiplayer.
